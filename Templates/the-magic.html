The Magic

How it Works

1. Hand-Picked Categorical Information

After a long exploration period we agreed on a strategy that would enable us to harness the features of beer styles, while also injecting unique characteristics of different beers within those styles.

Unfortunately, that meant we had to do some manual work upfront to classify the different beer styles with categorical information. As a group, we determined a handful of characteristics for each beer style, namely: color, flavor, hoppiness, maltiness, fruitiness, body, sweetness, sour, spicy, bitter, aroma, and custom flavor.

2. Scrape, Scrape, and Scrape Some More

Scrape BeerAdvocate (sorry, BA) to build a beer database, since this somehow does not exist online in any meaningful fashion. Also scrape BeerAdvocate for User Reviews, as this is what we will eventually use as data for our machine learning algorithm to make recommendations. Again, sorry, BA.

3. One Hot Encoding, the Pandas Way

Use one hot encoding to assign values to our categorical information including color, aroma, and custom flavor. Instead of using the SciKit Learn way, we flexed our Pandas skills and used dummy values to one hot encode our features.

4. A Little Machine Learning Powder

Test machine learning algorithms. And continue testing. And test some more. We started with K-Nearest Neighbors and had good results, but had difficulty automating the selection of the optimal K-value. We experimented with a handful of other machine learning methods before decoding on Random Forest as our algorithm of choice.

This is only the half of it. We had to account for Users without enough reviews, and for users for which our algorithm did not give any recommendations (drink more beer, everyone). All in all, we found a method that delivers great results.

5. Drumroll, Please

Put everything together, and we have an interactive site which takes in a BeerAdvocate username, scrapes the site in real time for that users reviews, and then runs a machine learning algorithm to determine which beers that user might like. We eliminate the beers the user has already rated from our database, so as not to recommend a beer the user has already had, and then return a list of beer recommendations with high confidence.

Voila.
